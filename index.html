新建商品

配置商品类型、基础信息、商品素材、SKU、履约方式、CDKey 码池 ID 和初始库存。商品创建后，后续库存调整统一前往库存列表维护。

基础信息

本期仅支持虚拟装扮、CDKEY、实物三个一级类型。
由商品类型自动确定，不支持单独修改。
商品对内名称建议使用英文下划线格式，例如 avatar_frame_s1、pubg_gcoin_300。
请选择商品类型，系统将加载当前已上架的 SKU 属性模板；如存在 SPU 属性模板，将同步展示到基础信息中。

SPU 运营展示配置

兑换量展示基数仅用于前台展示，前台展示兑换数量 = 真实兑换数量 + 兑换量展示基数；不参与订单统计、库存扣减和履约处理。

商品素材

*商品头图
用于商品主图展示，一套素材适配 PC / APP，支持 1 张。
通用素材
*商品详情图
一套素材适配 PC / APP，支持 webp / png / jpg，单张不超过 500K，尺寸比例 3:2，最多 10 张，可调整排序。
通用素材
用于商品详情页图文说明，支持富文本编辑，可配置权益介绍、使用规则、注意事项等内容。
用于说明兑换后的到账、使用、有效期或售后注意事项，非必填。
本期商品素材包含商品头图、商品详情图、商品说明和兑换说明；商品头图和商品详情图均使用一套通用素材适配 PC / APP，商品说明和兑换说明使用富文本配置。

适用范围与标签

适用范围与标签用于商品级运营识别、前台投放标识和服务端过滤，描述性标签输入后按空格生成标签块，同时兼容逗号或点号分割；商品角标标签通过下拉选择。商品是否前台投放仍由商品投放管理中的投放关系决定。

*初始库存配置

首次新建商品时支持配置 SKU 初始库存；履约方式为自动履约（CDKey）时，库存模式固定为限量，初始总库存由绑定的 CDKey 码池同步，不支持在商品新建页手动设置。
非 CDKey 商品的初始库存只用于商品首次创建时初始化库存记录。CDKey 商品绑定码池 ID 后，库存数据以 CDKey 码池列表为唯一来源，保证库存唯一性和正确性。

商品上架不等于前台投放。全局模板只能创建 1 个，投放范围固定为全部社区；游戏专区模板可以创建多个，并通过社区绑定控制投放范围。商城客户端可以在同一社区内投放多个 TAB 分区；每个 TAB 对应一条投放模板。同一商品可以投放到多个二级模板，并在各二级模板下分别维护排序。投放管理只维护模板与商品的投放关系，不维护商品基础信息、SKU 属性模板、库存、CDKey 码池和履约记录。

下架属性模板

下架后，该模板不可再被商品新建 / 编辑页引用；已创建商品的历史投放、兑换、订单和履约记录不受影响。

删除属性模板

删除后该模板将从常规列表中移除，商品新建 / 编辑页不可引用，并记录删除人、删除时间和删除原因。删除不是模板状态。

复制属性模板
复制模板会复制当前模板的属性项，新模板默认为草稿状态。复制后如需上架，需要先补全必填信息。
删除商品

删除后该商品将在后台常规列表和前台接口中不可见，并记录删除人、删除时间和删除原因。删除不是商品状态，不新增“已删除”状态。

下架商品

下架后商品不可前台投放、不可兑换，但历史订单、库存流水和履约记录不受影响。
